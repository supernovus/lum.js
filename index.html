Lum[V5] Test

Testing must be done via the JS console. The global-object compatibility library is available as the Lum global variable, and a custom library testing helper is available as the lt global variable.

Use lt(...libs) to load any other libraries you might want. If you want a full list, lt() will give that to you. Omit the directory, and the .js extension, as it's supplied for you.

Use lt.lum(...libs) to load more Lum.js libraries, which is anything in the scripts/nano folder. Omit the .js file extension, it's supplied for you.

In addition to the main libraries in scripts/nano, there are also several external scripts available in scripts/ext that can be loaded using lt.ext(...libs):

clipboard, easel, feature, jquery, jquery-ui, luxon, moment, require, sprintf, stacktrace.

The Crypto-JS libraries can be loaded using lt.cjs(...libs); if you don't know the exact library, omitting any arguments will load the `crypto-js` bundle which includes every major library in one file.

The ACE Editor can be loaded using lt.ace(); no arguments needed, as further ACE features are loaded on-demand, using an package-local version of RequireJS that won't conflict with the version provided by lt.ext().
